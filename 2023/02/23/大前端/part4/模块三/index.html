React Hooks

React Hooks

hooks是用来干什么的

是对函数组件的增强，让函数组件可以存储状态，拥有处理副作用的能力，在不使用类组件的情况下实现相同的功能。

什么是副作用

只要不是把数据转换成视图的代码就是副作用，eg: 发送请求，获取元素给元素添加事件，设置定时器等

类组件的不足（hooks要解决的问题）

  • 缺少逻辑复用机制：只能使用高阶组件，组件层级增加，调试复杂。
  • 类组件经常会变得复杂难以维护：同一个业务逻辑要拆分到多个生命周期中写多次，一个生命周期中包含多个不相干的业务。
  • 类成员方法不能保证this指向的正确性：需要bind绑定、或者函数嵌套函数。

useState

  • 为函数组件引入状态，用闭包来实现保存状态的。
  • useState要传入一个默认值，默认值可以是任何类型。可以是函数，函数返回什么初始值就是什么，函数有一个形参是count，函数只会调用一次，用于初始值是动态的情况。

设置值的方法是异步的。当下设置之后打印数据的话，数据还是上一次的数据，解决，需要把设置的代码写在回调函数中。

  • 返回的是一个数组，数组第一个元素是保存的状态，第二个是设置状态的方法，使用set开头加上状态名称。
  • 可以多次调用useState用来保存不同的状态。

useReducer是另一种让函数组件保存状态的方式

相比于useState的好处，比如app组件中的某个子组件需要改变count的值，不需要给子组件传递多个修改的方法，只需要把dispatch传递给子组件即可，子组件只需要调用dispatch即可。

useContent:跨组件层级获取数据时简化获取数据代码

useEffect: 处理副作用

  • 可以把相关业务逻辑都放在一起。
  • 可以把 useEffect 看做componentDidMount, componentDidUpdate 和 componentWillUnmount 这三个函数的组合

useEffect(() => {}) => componentDidMount, componentDidUpdate 组件挂在完成之后执行，组件数据更新完成后执行
useEffect(() => {}, []) => componentDidMount 组件挂在完成之后执行
useEffect(() => () => {}) => componentWillUnMount 组件卸载之前执行

useEffect中的参数函数不能是异步函数,因为useEffect函数要返回清理资源的函数,如果是异步函数就变成了返回Promise

useMemo:

  • useMemo 的行为类似Vue中的计算属性,可以监测某个值的变化,根据变化值计算新值。
  • useMemo 会缓存计算结果。如果监测值没有发生变化，即使组件重新染，也不会重新计算，此行为可以有助于避免在每个染上进行昂贵的计算

memo 方法

memo方法用来性能优化,如果本组件重新渲染之前，组件中的数据没有发生变化，就阻止组件更新。类似类组件中的 PureComponent 和 shouldComponentupdate
memo方法的参数是需要优化的组件，返回结果是一个新的组件。

useCallback(): 用来性能优化,缓存函数,使组件重新渲染时得到相同的函数实例.
